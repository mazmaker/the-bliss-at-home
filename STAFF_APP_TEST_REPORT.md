## ขั้นตอนการทดสอบที่แนะนำ

### 1. การเข้าถึงเว็บไซต์ ✅
- เปิดเบราว์เซอร์และเข้า https://wallet-rfc-kelly-part.trycloudflare.com
- **ผลลัพธ์:** แอปพลิเคชันโหลดได้ปกติ แสดง "Bliss Provider - The Bliss Massage at Home"

### 2. การทดสอบข้อมูลงาน (ต้องทดสอบจริง)
**ขั้นตอนที่ต้องตรวจสอบ:**
- [ ] ตรวจสอบว่ามีการจองที่มีสถานะ "ASSIGNED" หรือ "CONFIRMED" 
- [ ] ตรวจสอบว่าแสดงข้อมูลลูกค้า ที่อยู่ และรายละเอียดงานครบถ้วน
- [ ] ตรวจสอบการแสดงจำนวนเงินที่จะได้รับ (ต้องไม่ใช่ 0 บาท)

### 3. การทดสอบปุ่ม "เริ่มเดินทาง" 🔍
**ฟีเจอร์ที่ต้องทดสอบ:**
- [ ] ปุ่มสีเขียว "เริ่มเดินทาง (ติดตาม GPS)" แสดงได้
- [ ] กดปุ่มแล้วระบบขออนุญาต GPS Location
- [ ] หลังอนุญาตแล้ว ปุ่มเปลี่ยนเป็น "กำลังติดตาม GPS" พร้อมจุดสีเขียวกระพริบ
- [ ] แสดงข้อความ "ลูกค้าเห็นตำแหน่งของคุณแล้ว"
- [ ] แสดงเวลาอัพเดทล่าสุดของ GPS

### 4. การทดสอบปุ่ม "มาถึงแล้ว" 🔍
**ขั้นตอนต่อจาก GPS:**
- [ ] ปุ่มสีน้ำเงิน "มาถึงแล้ว" แสดงขึ้นระหว่างติดตาม GPS
- [ ] กดปุ่มแล้วการติดตาม GPS หยุด
- [ ] ปุ่มเปลี่ยนเป็น "เริ่มงาน" สีน้ำเงิน

### 5. การทดสอบปุ่ม "เริ่มงาน" ⭐
**ขั้นตอนสำคัญ:**
- [ ] กดปุ่ม "เริ่มงาน" หลังจาก GPS เสร็จ
- [ ] สถานะงานเปลี่ยนเป็น "กำลังดำเนินการ" (in_progress)
- [ ] **Service Timer** แสดงขึ้นด้านบน พร้อมนับเวลาจริง
- [ ] Timer แสดงรูปแบบ "MM:SS" หรือ "นาที" ไม่ใช่ 0 นาที

### 6. การทดสอบ Service Timer ⭐⭐
**ฟีเจอร์สำคัญที่ต้องทำงาน:**
- [ ] **เวลาเดินทาง** - แสดงเวลาจากเริ่มติดตาม GPS จนถึงเริ่มงาน
- [ ] **เวลาให้บริการ** - แสดงเวลาปัจจุบันตั้งแต่เริ่มงาน (นับเป็นวินาทีแบบ Real-time)
- [ ] **เวลาคงเหลือ** - แสดงเวลาที่เหลือจากระยะเวลาที่กำหนด
- [ ] สีเปลี่ยนเมื่อใกล้หมดเวลา (เหลือ < 5 นาที = เหลือง, เกินเวลา = แดง)

### 7. การทดสอบ Billing System ⭐⭐
**การคำนวณเงินที่ต้องถูกต้อง:**
- [ ] **ค่าชดเชยการเดินทาง** - 5 บาท/นาที สูงสุด 150 บาท
- [ ] **ค่าบริการ** - 30 บาท/นาที (คำนวณจากเวลาจริงที่ให้บริการ)
- [ ] **รวมทั้งหมด** - แสดงยอดรวมแบบ Real-time เปลี่ยนตามเวลา
- [ ] จำนวนเงินต้อง**ไม่เป็น 0 บาท**และเพิ่มขึ้นทุกนาที

### 8. การทดสอบฟีเจอร์ GPS อื่น ๆ 🔍
**ฟีเจอร์เพิ่มเติม:**
- [ ] ปุ่ม "แชร์" - สร้างลิงก์ติดตามให้ลูกค้า
- [ ] ปุ่ม "โทร" - เปิดแอปโทรศัพท์ (ถ้ามีเบอร์ลูกค้า)
- [ ] ปุ่ม Map icon - เปิด Google Maps สำหรับนำทาง
- [ ] คำเตือน "ห้ามปิดแอป • ห้ามล็อคหน้าจอ • เสียบชาร์จ"

### 9. การทดสอบปุ่ม "เสร็จสิ้นงาน" ⭐
**ขั้นตอนสุดท้าย:**
- [ ] ปุ่มสีเขียว "เสร็จสิ้นงาน" แสดงระหว่างให้บริการ
- [ ] กดปุ่มแล้วTimer หยุด
- [ ] แสดงยอดเงินสุดท้ายที่จะได้รับ
- [ ] สถานะเปลี่ยนเป็น "เสร็จสิ้น" (completed)

## จุดสำคัญที่ต้องเช็ค ⚠️

### จุดเสี่ยง Console Errors:
- เปิด Developer Console (F12) ตรวจสอบ JavaScript errors
- GPS permission errors
- Supabase connection errors
- Authentication errors

### จุดเสี่ยงการคำนวณ:
- Timer ติดที่ 0 นาที = **Bug ร้ายแรง**
- Billing แสดง 0 บาท = **Bug ร้ายแรง** 
- GPS ไม่ขออนุญาต = **Bug GPS**
- Timer ไม่นับเวลา = **Bug Timer**

## ขั้นตอนที่แนะนำสำหรับการทดสอบ

1. **เตรียมข้อมูล** - ให้แน่ใจว่ามีการจองใน database ที่มีสถานะ ASSIGNED/CONFIRMED
2. **ทดสอบทีละขั้นตอน** - อย่าข้ามขั้นตอน เพราะแต่ละขั้นตอนขึ้นอยู่กับขั้นตอนก่อนหน้า
3. **จับเวลา** - ให้เวลาระบบประมวลผล 2-3 วินาทีหลังกดปุ่มแต่ละครั้ง
4. **เช็ค Console** - เปิด Developer Tools ตลอดเวลาเพื่อดู errors
5. **ทดสอบใน Mobile Browser** - ใช้ Chrome Mobile หรือ Safari บนมือถือ

## สิ่งที่คาดหวัง ✅

### การทำงานที่ถูกต้อง:
- [ ] **GPS tracking ใช้งานได้** - ขออนุญาต location และติดตามได้
- [ ] **State transitions ถูกต้อง** - assigned → traveling → arrived → in_progress → completed
- [ ] **Timer นับเวลาจริง** - แสดงนาที:วินาที และเปลี่ยนแปลงทุกวินาที
- [ ] **Billing คำนวณถูก** - 30 บาท/นาที + ค่าเดินทาง
- [ ] **UI responsive** - ใช้งานได้บนมือถือ

### สิ่งที่ไม่ควรเกิดขึ้น ❌:
- [ ] Timer แสดง "0 นาที" ตลอดเวลา
- [ ] Billing แสดง "0 บาท" ตลอดเวลา  
- [ ] ปุ่ม GPS ไม่ทำงาน/ไม่เปลี่ยนสถานะ
- [ ] JavaScript errors ใน console
- [ ] แอปค้างหรือไม่ตอบสนอง
