## ⭐ การทดสอบหลัก (ตามลำดับ)

### 🚗 3. ทดสอบปุ่ม "เริ่มเดินทาง"
**คาดหวัง:** ปุ่มสีเขียว "เริ่มเดินทาง (ติดตาม GPS)"

- [ ] กดปุ่ม "เริ่มเดินทาง"
- [ ] เบราว์เซอร์ขออนุญาต GPS Location → **กดอนุญาต**
- [ ] ปุ่มเปลี่ยนเป็น "กำลังติดตาม GPS" พร้อมจุดสีเขียวกระพริบ
- [ ] แสดงข้อความ "ลูกค้าเห็นตำแหน่งของคุณแล้ว"

**ถ้าไม่ทำงาน:** ตรวจสอบ Console (กด F12) หา GPS errors

### 📍 4. ทดสอบปุ่ม "มาถึงแล้ว"  
**คาดหวัง:** ปุ่มสีน้ำเงิน "มาถึงแล้ว" แสดงระหว่างติดตาม GPS

- [ ] กดปุ่ม "มาถึงแล้ว"
- [ ] การติดตาม GPS หยุดทำงาน
- [ ] ปุ่มเปลี่ยนเป็น "เริ่มงาน" สีน้ำเงิน

### 🏃‍♂️ 5. ทดสอบปุ่ม "เริ่มงาน" (สำคัญมาก!)
**คาดหวัง:** ปุ่มสีน้ำเงิน "เริ่มงาน"

- [ ] กดปุ่ม "เริ่มงาน"
- [ ] สถานะเปลี่ยนเป็น "กำลังดำเนินการ"
- [ ] **Service Timer ปรากฏด้านบน** ← **นี่คือจุดสำคัญ!**

## ⏰ 6. ทดสอบ Service Timer (สำคัญมากที่สุด!)

**Timer ต้องแสดง:**
- [ ] **เวลาเดินทาง** - เวลาจากเริ่ม GPS จนเริ่มงาน (สีน้ำเงิน)
- [ ] **เวลาให้บริการ** - เวลาปัจจุบันนับจากเริ่มงาน
- [ ] **เวลาคงเหลือ** - เวลาที่เหลือจากการจองต้นฉบับ

**การทดสอบ:**
- [ ] เวลาเปลี่ยนทุกวินาที (ไม่ใช่แค่ทุกนาที)
- [ ] รูปแบบ: "MM:SS" หรือ "X นาที"  
- [ ] **ห้ามแสดง "0 นาที" ตลอดเวลา** ← Bug ร้ายแรง!

## 💰 7. ทดสอบ Billing System (สำคัญมากที่สุด!)

**ต้องแสดง:**
- [ ] **ค่าชดเชยเดินทาง** - 5 บาท/นาที (สูงสุด 150 บาท)
- [ ] **ค่าบริการ** - 30 บาท/นาที จากเวลาจริงที่ให้บริการ
- [ ] **ยอดรวม** - เพิ่มขึ้นทุกนาทีตามเวลาจริง

**การทดสอบ:**
- [ ] รอ 1-2 นาที แล้วดูว่ายอดเงินเปลี่ยน
- [ ] คำนวณ: หากให้บริการ 5 นาที = 5 x 30 = 150 บาท + ค่าเดินทาง
- [ ] **ห้ามแสดง "0 บาท" ตลอดเวลา** ← Bug ร้ายแรง!

## ✅ 8. ทดสอบปุ่ม "เสร็จสิ้นงาน"
**คาดหวัง:** ปุ่มสีเขียว "เสร็จสิ้นงาน" แสดงระหว่างให้บริการ

- [ ] กดปุ่ม "เสร็จสิ้นงาน"
- [ ] Timer หยุดนับ
- [ ] แสดงยอดเงินสุดท้าย
- [ ] สถานะเปลี่ยนเป็น "เสร็จสิ้น"

## 🚨 จุดเสี่ยงที่ต้องระวัง

### ❌ Bug ร้ายแรงที่ห้ามเกิด:
1. **Timer แสดง 0 นาที ตลอดเวลา**
2. **Billing แสดง 0 บาท ตลอดเวลา**
3. **GPS ไม่ขออนุญาต location**
4. **ปุ่มกดแล้วไม่เปลี่ยนสถานะ**

### 🔍 วิธีตรวจสอบ Error:
1. **กด F12** เปิด Developer Console
2. ดู tab **Console** หา error สีแดง
3. ดู tab **Network** ตรวจสอบ API calls ที่ล้มเหลว

### 📱 ทดสอบบนมือถือ:
- เปิดด้วย Chrome หรือ Safari บนมือถือ
- GPS ควรทำงานได้ดีกว่าบน Desktop
- จอสัมผัสควรตอบสนองได้ดี

### 🟢 สำเร็จ หมายถึง:

- ปุ่มกดแล้วไม่มีการตอบสนอง
