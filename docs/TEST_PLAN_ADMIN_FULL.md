### MODULE 1: Authentication

---

#### TC-001: Login — เข้าสู่ระบบสำเร็จ

**วัตถุประสงค์:** ตรวจสอบว่า Admin สามารถ login ด้วย email/password ที่ถูกต้องได้

**ขั้นตอน:**
1. เปิดหน้า `/admin/login`
2. กรอก email: `admintest@theblissathome.com`
3. กรอก password: `Admin@12345`
4. กดปุ่ม "เข้าสู่ระบบ"

**ผลลัพธ์ที่คาดหวัง:**
- [ ] แสดงหน้า Login พร้อม form email/password
- [ ] กดเข้าสู่ระบบ → redirect ไปหน้า Dashboard (`/admin`)
- [ ] แสดงชื่อผู้ใช้/avatar ที่ header

---

2. กรอก email: `wrong@email.com`
3. กรอก password: `Admin@12345`
4. กดปุ่ม "เข้าสู่ระบบ"

- [ ] แสดง error message (เช่น "อีเมลหรือรหัสผ่านไม่ถูกต้อง")
- [ ] ไม่ redirect ไปหน้าอื่น

#### TC-006: Logout — ออกจากระบบ

**วัตถุประสงค์:** ตรวจสอบว่า logout ทำงานถูกต้อง

#### TC-007: Session Persistence — refresh แล้วยังล็อกอินอยู่

**ผลลัพธ์ที่คาดหวัง:**

- [ ] ไม่ redirect ไป Login
- [ ] ข้อมูลผู้ใช้ยังแสดงอยู่

#### TC-008: Protected Route — เข้าหน้า protected โดยไม่ login

**ขั้นตอน:**
1. ไม่ login (ใช้ incognito หรือ clear session)
2. เปิด URL `/admin/staff` โดยตรง

**ผลลัพธ์ที่คาดหวัง:**
- [ ] Redirect ไปหน้า `/admin/login`
- [ ] ไม่แสดงข้อมูลใด ๆ ของระบบ

### MODULE เพิ่มเติม: Cross-Module / Edge Cases

---

#### TC-154: Responsive — หน้าจอมือถือ

**วัตถุประสงค์:** ตรวจสอบว่า Admin app แสดงผลบนหน้าจอเล็กได้

**ขั้นตอน:**
1. Resize browser เป็นขนาดมือถือ (375px)
2. ตรวจสอบ sidebar/navigation

**ผลลัพธ์ที่คาดหวัง:**
- [ ] Sidebar collapse เป็น hamburger menu
- [ ] เนื้อหาไม่ overflow ออกนอกจอ
- [ ] สามารถใช้งานได้ทุกหน้า

---

#### TC-155: Navigation — Sidebar Menu

**วัตถุประสงค์:** ตรวจสอบว่า sidebar menu ทำงานถูกต้อง

**ขั้นตอน:**
1. คลิก menu แต่ละรายการใน sidebar

**ผลลัพธ์ที่คาดหวัง:**
- [ ] คลิก → navigate ไปหน้าที่ถูกต้อง
- [ ] Menu item ที่ active แสดง highlight
- [ ] ทุก menu item ทำงาน (ไม่มี broken link)

---

#### TC-156: Loading State — ทุกหน้า

**วัตถุประสงค์:** ตรวจสอบว่าทุกหน้าแสดง loading state ระหว่างโหลดข้อมูล

**ขั้นตอน:**
1. เปิดแต่ละหน้าและสังเกตช่วงโหลด

**ผลลัพธ์ที่คาดหวัง:**
- [ ] แสดง loading spinner / skeleton ระหว่างโหลด
- [ ] ไม่แสดงหน้าว่างหรือ error ระหว่างโหลด

---

#### TC-157: Error State — Network Error

**วัตถุประสงค์:** ตรวจสอบว่าแสดง error state เมื่อ network ล้มเหลว

**ขั้นตอน:**
1. ปิด internet / block API request
2. พยายามโหลดหน้า

**ผลลัพธ์ที่คาดหวัง:**
- [ ] แสดง error message ที่เข้าใจง่าย
- [ ] มีปุ่ม "ลองใหม่" / "Retry"
- [ ] ไม่แสดง error stack trace

---

#### TC-158: Pagination — ทุกหน้ารายการ

**วัตถุประสงค์:** ตรวจสอบว่า pagination ทำงานถูกต้อง

**ขั้นตอน:**
1. เปิดหน้าที่มีข้อมูลเยอะ (Bookings, Staff, Customers)
2. กดเปลี่ยนหน้า

**ผลลัพธ์ที่คาดหวัง:**
- [ ] เปลี่ยนหน้าได้
- [ ] ข้อมูลแต่ละหน้าไม่ซ้ำกัน
- [ ] แสดงหน้าปัจจุบัน / จำนวนทั้งหมด

---

#### TC-159: URL Direct Access — ทุก Route

**วัตถุประสงค์:** ตรวจสอบว่าเข้าแต่ละ route ผ่าน URL ตรงได้ (หลัง login)

**ขั้นตอน:**
1. Login แล้ว
2. พิมพ์ URL แต่ละหน้าโดยตรง (เช่น `/admin/staff`, `/admin/bookings`)

**ผลลัพธ์ที่คาดหวัง:**
- [ ] เข้าถึงหน้านั้นได้ตรง
- [ ] ไม่ redirect กลับ Dashboard

---

#### TC-160: 404 Page — Route ที่ไม่มี

**วัตถุประสงค์:** ตรวจสอบว่าแสดง 404 page สำหรับ route ที่ไม่มี

**ขั้นตอน:**
1. เข้า URL ที่ไม่มีอยู่ (เช่น `/admin/nonexistent`)

**ผลลัพธ์ที่คาดหวัง:**
- [ ] แสดง 404 page หรือ redirect กลับ Dashboard
- [ ] ไม่แสดง blank page / error
