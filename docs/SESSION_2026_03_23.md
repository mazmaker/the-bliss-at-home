## Key RLS Patterns
1. **ห้ามใช้ `FOR ALL`** — ใช้แยก SELECT/INSERT/UPDATE/DELETE
2. **UPDATE ต้องมีทั้ง `USING` + `WITH CHECK`**
3. **dependent table access** — ถ้า RLS ของ table A query table B ต้อง grant SELECT บน B
4. **`PGRST116` + 0 rows** → RLS blocking, ไม่ใช่ data missing
5. **`bookings`, `customers`, `staff`, `services`** มี **RLS ปิด** (`rowsecurity = false`) ใน DB ที่ restore
