### PART A: Authentication (การเข้าสู่ระบบ)

---

#### TC-01: Login — เข้าสู่ระบบสำเร็จ

**วัตถุประสงค์:** ตรวจสอบว่า Hotel user สามารถ login ด้วย email/password ที่ถูกต้องได้

**ขั้นตอน:**
1. เปิดหน้า Hotel Login (`/login`)
2. กรอก Email: `reservations@hilton.com`
3. กรอก Password: `Hotel123.`
4. กดปุ่ม "เข้าสู่ระบบ"

**ผลลัพธ์ที่คาดหวัง:**
- [ ] หน้า Login แสดงฟอร์ม email + password
- [ ] Login สำเร็จ → redirect ไปหน้า Dashboard (`/hotel/:slug`)
- [ ] แสดงชื่อโรงแรมใน layout/header

---

#### TC-02: Login — รหัสผ่านผิด

**วัตถุประสงค์:** ตรวจสอบว่าระบบแสดง error เมื่อกรอกรหัสผ่านผิด

**ขั้นตอน:**
1. เปิดหน้า Hotel Login
2. กรอก Email: `reservations@hilton.com`
3. กรอก Password: `WrongPassword123`
4. กดปุ่ม "เข้าสู่ระบบ"

**ผลลัพธ์ที่คาดหวัง:**
- [ ] แสดง error message (เช่น "อีเมลหรือรหัสผ่านไม่ถูกต้อง")
- [ ] ไม่ redirect ไปหน้าอื่น
- [ ] ยังอยู่หน้า Login

---

#### TC-03: Login — อีเมลไม่มีในระบบ

---

#### TC-04: Login — ฟอร์ม validation (ช่องว่าง)

**วัตถุประสงค์:** ตรวจสอบว่าฟอร์มไม่ให้ submit เมื่อไม่กรอกข้อมูล

**ขั้นตอน:**
1. เปิดหน้า Hotel Login
2. ไม่กรอกอะไร → กดปุ่ม "เข้าสู่ระบบ"
3. กรอก Email อย่างเดียว → กดปุ่ม "เข้าสู่ระบบ"
4. กรอก Password อย่างเดียว → กดปุ่ม "เข้าสู่ระบบ"

**ผลลัพธ์ที่คาดหวัง:**
- [ ] ไม่กรอกทั้งสอง → แสดง validation error หรือไม่ submit
- [ ] กรอก Email อย่างเดียว → แสดง validation error ที่ช่อง Password
- [ ] กรอก Password อย่างเดียว → แสดง validation error ที่ช่อง Email

---

#### TC-05: Logout — ออกจากระบบ

**วัตถุประสงค์:** ตรวจสอบว่า Hotel user สามารถ logout ได้

**ขั้นตอน:**

#### TC-06: Session Persistence — จำ session หลัง refresh

**วัตถุประสงค์:** ตรวจสอบว่า session ยังอยู่หลังจาก refresh หน้า

**ผลลัพธ์ที่คาดหวัง:**
- [ ] ยังอยู่หน้า Dashboard (ไม่ต้อง login ใหม่)
- [ ] ข้อมูลโรงแรมยังแสดงถูกต้อง

---

#### TC-07: Protected Routes — เข้าหน้าที่ต้อง login โดยไม่ได้ login

**วัตถุประสงค์:** ตรวจสอบว่า route ที่ต้อง login จะ redirect ไป Login page

**ขั้นตอน:**
1. ไม่ได้ login (หรือ logout แล้ว)
2. พยายามเข้าหน้า Dashboard โดยตรง (`/hotel/:slug`)
3. พยายามเข้าหน้า Services (`/hotel/:slug/services`)
4. พยายามเข้าหน้า Booking History (`/hotel/:slug/history`)

**ผลลัพธ์ที่คาดหวัง:**
- [ ] ทุกหน้า → redirect ไปหน้า Login
- [ ] ไม่แสดงข้อมูลใดๆ ก่อน redirect

- [ ] ราคารวม = ราคาเดิม + ราคา extension

- [ ] Booking อัปเดต duration ใหม่

#### TC-99: Extend Session — Validation ไม่เลือก duration

#### TC-100: Extend Session — Error Handling (extension ล้มเหลว)

- [ ] ไม่ crash
